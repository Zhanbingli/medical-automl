# OSF Submission Checklist

## Before Upload

- Use the public manuscript PDF at [docs/li_2026_clinical_prediction_schema_shift_preprint.pdf](/Users/lizhanbing12/contri_pt/autoresearch-macos/docs/li_2026_clinical_prediction_schema_shift_preprint.pdf:1)
- Use a stable filename such as `li_2026_clinical_prediction_schema_shift_preprint.pdf`
- Verify the title in the PDF matches [docs/osf_preprint_metadata.md](/Users/lizhanbing12/contri_pt/autoresearch-macos/docs/osf_preprint_metadata.md:1)
- Confirm author name, affiliation, ORCID, and email are correct
- Skim the abstract one last time for wording you are willing to defend publicly
- Make sure the GitHub repo is public or ready to be made public

## Suggested Upload Bundle

- Main file: manuscript PDF
- Optional supplement 1: repository link in the metadata rather than a separate file
- Optional supplement 2: a small ZIP with figures only if you want OSF readers to download them directly
- Optional supplement 3: cover note describing this as a preprint case study in clinical AI reliability

## OSF Form Fields

- Title: use the exact title from [docs/osf_preprint_metadata.md](/Users/lizhanbing12/contri_pt/autoresearch-macos/docs/osf_preprint_metadata.md:5)
- Abstract: paste from [docs/osf_preprint_metadata.md](/Users/lizhanbing12/contri_pt/autoresearch-macos/docs/osf_preprint_metadata.md:9)
- Authors: add yourself and ORCID
- Subjects: machine learning / AI / health informatics / medicine
- License: pick one option and keep it consistent
- Supplemental materials: optional

## Recommended Release Order

1. Finalize the manuscript PDF
2. Push the cleaned GitHub repo
3. Create the OSF preprint record
4. Upload the PDF and paste the metadata
5. Save the DOI and public URL
6. Add the DOI back into `README.md`

## After Upload

- Add the OSF DOI to the README citation block
- Add the OSF link to your CV, email signature, and any cold emails
- If you later secure arXiv endorsement, upload a revised version there and cross-link both records

## Do Not Forget

- OSF is your immediate public-proof route; do not wait for arXiv endorsement first
- Keep the repo wording aligned with the preprint wording
- Treat this as a versioned public artifact; update with a new version instead of silently replacing major claims
